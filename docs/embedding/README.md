# モデリング・ベクトル化

このディレクトリでは、テキストデータを**意味を持つ数値ベクトル（分散表現）**に変換し、高度な分析を行うための手法を扱います。

## モデリングとは

形態素解析によって得られた「単語のリスト」を、機械学習を用いて「数値の列（ベクトル）」に変換するプロセスです。
単なる出現頻度の集計とは異なり、単語の**「意味」や「文脈」**を捉えることができるのが最大の特徴です。

### BoW / TF-IDF との違い

| 特徴 | BoW / TF-IDF (`docs/bow`) | モデリング / Embedding (`docs/embedding`) |
| :--- | :--- | :--- |
| **基本原理** | 単語の**出現回数**を数える | 単語の**使われる文脈**を学習する |
| **表現形式** | **疎ベクトル (Sparse)**<br>0 が多く、次元数は語彙数と同じ（数万〜） | **密ベクトル (Dense)**<br>実数が詰まっており、次元数は低い（100〜300） |
| **「意味」** | **捉えられない**<br>「車」と「自動車」は別の単語として扱う | **捉えられる**<br>「車」と「自動車」は近いベクトルになる |
| **主な用途** | キーワード抽出、検索、文書分類 | 類義語探索、意味の演算、クラスタリング |

## 手法の選び方

目的に応じて、適切な手法（ドキュメント）を選択してください。

| 手法 | 対象 | 特徴 | リンク |
| :--- | :--- | :--- | :--- |
| **Word2Vec** | 単語 | 単語の意味をベクトル化。類義語検索や演算が可能。 | [`word2vec.md`](./word2vec.md), [`word2vec_example_workflow.md`](./word2vec_example_workflow.md)|
| **Doc2Vec** | 文書 | 文書全体をベクトル化。類似文書検索やタグ推定に利用。 | [`doc2vec.md`](./doc2vec.md) |

## 前提知識

これらのモデリングを行うには、大量のテキストデータが必要です。
本リポジトリでは、メモリ不足を防ぐために **`CorpusDB`** を使用したストリーミング学習を推奨しています。
